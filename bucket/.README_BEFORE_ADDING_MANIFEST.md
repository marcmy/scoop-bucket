# 🚨 READ THIS BEFORE ADDING OR EDITING A MANIFEST 🚨

## GitHub API authentication is not optional in Actions

GitHub-hosted runners share public IP addresses. An unauthenticated request to
`api.github.com` can therefore hit the shared 60-requests-per-hour IP limit even
when this repository's workflow token still has its full authenticated quota.

A failed run may succeed immediately when retried on another runner. That does
**not** mean the manifest is fixed.

`GITHUB_TOKEN` is generated automatically for every GitHub Actions job. It does
not appear under the repository's manually configured Actions secrets. The
workflow exposes it to Scoop and custom manifest scripts as the
`GITHUB_TOKEN` environment variable.

## Use the right pattern

### 1. Preferred: let Scoop fetch the GitHub API URL

When `checkver.url` itself points to `https://api.github.com/...`, Scoop detects
the GitHub API URL and adds authentication automatically when a token is
available.

```json
"checkver": {
    "url": "https://api.github.com/repos/OWNER/REPO/releases/latest",
    "jsonpath": "$.tag_name"
}
```

A `checkver.script` that only parses `$page` also uses the response Scoop already
downloaded and does not need to make another request.

### 2. Custom API request: add conditional authentication

When a script calls `Invoke-RestMethod` or `Invoke-WebRequest` itself, Scoop
cannot inject authentication into that internal request. Build the header
without authorization, then add authorization only when the token exists:

```powershell
$headers = @{
    'User-Agent' = 'Scoop-Checkver'
    'Accept' = 'application/vnd.github+json'
    'X-GitHub-Api-Version' = '2022-11-28'
}

if ($env:GITHUB_TOKEN) {
    $headers['Authorization'] = "Bearer $env:GITHUB_TOKEN"
}

$response = Invoke-RestMethod -Headers $headers -Uri 'https://api.github.com/repos/OWNER/REPO/...'
```

Inside a JSON manifest, remember to escape the double quotes:

```json
"script": [
    "$headers=@{ 'User-Agent'='Scoop-Checkver'; 'Accept'='application/vnd.github+json'; 'X-GitHub-Api-Version'='2022-11-28' }",
    "if ($env:GITHUB_TOKEN) { $headers['Authorization'] = \"Bearer $env:GITHUB_TOKEN\" }",
    "$response=Invoke-RestMethod -Headers $headers -Uri 'https://api.github.com/repos/OWNER/REPO/...'"
]
```

## Never do either of these

### Do not make a custom GitHub API request anonymously

```powershell
Invoke-RestMethod -Uri 'https://api.github.com/repos/OWNER/REPO/...'
```

This creates intermittent Excavator failures from the shared hosted-runner IP
rate limit.

### Do not send an unconditional bearer header

```powershell
$headers['Authorization'] = "Bearer $env:GITHUB_TOKEN"
```

When the manifest is checked locally without `GITHUB_TOKEN`, this sends an empty
or invalid bearer token. Local use should fall back to a normal unauthenticated
request.

## Autoupdate hash URLs

An `autoupdate.hash.url` that points directly to `api.github.com` is fetched by
Scoop itself and normally uses Scoop's GitHub-token handling. Do not add a custom
request merely to wrap it unless the manifest genuinely requires extra logic.

## Workflow requirement

Any workflow that executes manifest checks must expose its automatically
generated token:

```yaml
env:
  GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

This repository currently does that in both Excavator and CI manifest-test
steps. Do not remove it because the repository has no manually added secret;
`secrets.GITHUB_TOKEN` is supplied automatically by GitHub Actions.

## Before committing a manifest

1. Search the changed manifest for `api.github.com`.
2. Search it for `Invoke-RestMethod` and `Invoke-WebRequest`.
3. If the script makes the request itself, verify conditional token handling.
4. Verify there is no unconditional `Authorization` header.
5. Validate the JSON.
6. Run the manifest check with a token.
7. Run it again locally without a token when practical.
8. Remember: a green rerun after a rate-limit failure is not proof of a fix.

Useful audit command:

```powershell
rg -n "api\.github\.com|Invoke-RestMethod|Invoke-WebRequest|GITHUB_TOKEN" bucket .github
```

## The one-sentence rule

> **Scoop-owned GitHub API fetch: Scoop handles auth. Script-owned GitHub API
> fetch: add `Authorization` conditionally when `GITHUB_TOKEN` exists.**
